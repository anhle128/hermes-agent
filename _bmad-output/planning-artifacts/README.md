# hermes-agent BMad Planning Handoff

This folder is the active local planning input package for isolated `hermes-agent` implementation.
Implementation agents should run from the `hermes-agent` subproject and should use the flat files in this folder as their implementation input.
The implementation root is the `hermes-agent` subproject root:

```text
hermes-agent/
```

Required local handoff files:

```text
README.md
prd.md
architecture.md
epics.md
contracts/workflow-commander/README.md
```

The flat `prd.md`, `architecture.md`, and `epics.md` files are the current implementation inputs.
They are ordinary local files, not symlinks.
They are written so implementation workflows do not need parent planning files.

Do not place implementation artifacts in this planning folder.
`hermes-agent` implementation artifacts are generated by workflows that run inside `hermes-agent`.

Candidate validation commands for downstream Hermes implementation work:

```text
uv sync --extra dev
uv run pytest
uv run ruff check .
```

The local contract package currently includes schema files under `contracts/workflow-commander/schemas/`.
The required example fixture families are still future work unless files are present under `contracts/workflow-commander/examples/`.
Downstream contract-gated stories must not treat required examples as available until those example files exist locally and pass compatibility validation.
